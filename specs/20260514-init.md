## Key root files

**`pnpm-workspace.yaml`**

```yaml
packages:
  - "apps/*"
  - "packages/*"
```

**`turbo.json`** вЂ” define `dev`, `build`, `lint`, `test`, `typecheck` pipelines. The `ai-service` exposes these via its wrapper `package.json` (`"dev": "uv run uvicorn app.main:app --reload"`), so `turbo dev` spins up everything including Python.

## A few notes worth flagging

The `ai-gateway` module inside NestJS is deliberate вЂ” keep all FastAPI calls behind a single typed boundary so the rest of your Nest code doesn't care that AI lives in a separate service. Same idea on the frontend with `@repo/api-client`: one place that knows about both backends.

For the Python service, I'd recommend **`uv`** over Poetry вЂ” it's much faster and plays well with Docker layer caching. Pin Python via `.python-version` so contributors don't drift.

Models (BGE embeddings, reranker) should be downloaded into `apps/ai-service/models/` by a script and gitignored вЂ” don't commit weights. The `scripts/download-models.sh` handles first-time setup.
